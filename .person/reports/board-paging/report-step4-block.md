# 페이징 블록(5개씩 끊어보기) 구현 완료 보고서

- 날짜: 2026-03-24
- 담당: Gemini CLI

## 1. 개요
사용자 경험(UX) 개선을 위해 전체 페이지 번호를 나열하던 기존 방식에서 5개씩 끊어서 보여주는 '페이지 블록' 방식으로 로직을 개선함.

## 2. 작업 상세 내역

### 🚀 BoardResponse.java
- **페이지 블록 계산 로직 도입**:
  - `blockSize`를 5로 설정.
  - 현재 페이지를 기반으로 소속된 블록을 계산(`currentBlock = (page - 1) / blockSize`).
  - 블록의 `startPage`와 `endPage`를 도출하여 `pageNumbers` 리스트에 현재 블록의 번호만 포함되도록 수정.
  - `endPage`가 전체 페이지 수(`totalPages`)를 초과하지 않도록 `Math.min` 처리.

## 3. 결과 요약
- **1-5, 6-10 등** 페이지 번호가 5개씩 묶여서 화면에 출력됨.
- 페이지가 아주 많아질 경우에도 하단 페이징 UI가 무너지지 않고 깔끔하게 유지됨.
- 0-index 기술적 한계와 1-index 사용자 친화적 처리가 완벽히 조화됨.

## 4. 확인 결과
- 예시: 7페이지를 보고 있다면 `[6, 7, 8, 9, 10]` 번호가 출력됨.
- 예시: 총 8페이지일 때 6페이지를 보고 있다면 `[6, 7, 8]` 번호가 출력됨.
